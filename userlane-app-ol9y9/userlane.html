You can directly test this on this page.

It seems like you opened this file directly from the file system. For the Userlane self-hosting package to work you need to put this folder and all it's files onto a webserver that serves these files via HTTP or HTTPS. Open this file (userlane.html) using the hostname of the webserver to get a ready-made version of the Snippet.

How to integrate it into the application

That really depends on the application. The only requirement for Userlane to be compatible is that it is a browser-based application that uses HTML, CSS, and JS for the interface.

Userlane should be generally loaded after the application has initialized. Usually the Snippet can be just put to the end of the page, just before the </body> tag. It needs to be reloaded after a page reload of course, so it is best practice to put it into the master template. On single page applications it only needs to be loaded once.
